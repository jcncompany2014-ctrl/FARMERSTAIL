# Voice & UX 가이드라인

**Last updated:** 2026-05-13
**Owner:** 솔로 운영자 (story@farmerstail.kr)

발명 명세서의 9개 모듈을 사용자 경험에 누가 되지 않게 적용하기 위한 카피·UX 정책. 모든 새 화면·메시지는 이 가이드라인을 따른다.

---

## 1. 단어 선택 — "신뢰도" 금지

| 금지 | 권장 |
|------|------|
| 신뢰도 72% | **맞춤도 72%** / **정밀 분석 진행률 72%** |
| 측정이 부정확합니다 | 더 정확하게 케어할 한 가지 방법이 있어요 |
| 사용자 점수 | 시스템 완성도 / 진행률 |

**핵심**: 사용자를 평가하는 게 아니라 시스템 완성도를 보여주는 톤. 결핍이 아닌 성장의 프레임.

---

## 2. 톤 — 견 주어 / 칭찬·격려

### 금지 표현
- "당신의 측정이 부정확합니다"
- "비만은 견주의 책임"
- "다른 견주는 평균 85%인데 당신은 72%"
- "강제로", "반드시"

### 권장 표현
- "초롱이 더 정확하게 케어할 한 가지 방법이 있어요" (견 주어)
- "함께 건강한 체중으로 가보아요" (책임 분산)
- "초롱이가 오늘 산책 잘 다녀왔네요" (칭찬)
- "초롱이를 위한 새로운 식단 제안이 있어요" (긍정)

### 어미
- 해요체 (~예요 / ~해요) 일관 사용
- 합쇼체 (~합니다) 는 법적 표기·약관·confirm 같은 공식 위치만

---

## 3. 비교의 금지

- **자기 자신과의 비교만 허용**: "지난주 대비 +8% 향상"
- **다른 견주와의 비교 금지**: "평균 85%, 당신은 72%" 같은 패턴 절대 X
- **자기 견의 시계열 변화만 시각화**: 12주 그래프

---

## 4. 부정적 정보의 다단계 노출

### 3단계 점진 공개 패턴

**1단계 (메인 메시지 — 모든 사용자에게 노출)**:
> "초롱이를 위한 새로운 식단을 제안해 드릴게요"

이유는 언급하지 않음. 사용자가 호기심 있으면 클릭.

**2단계 (사용자가 클릭 시)**:
> "현재 약간 이상 체중을 넘어선 상태예요. 새 식단으로 조정해보면 어떨까요?"

부드러운 표현으로 상황 안내.

**3단계 (사용자가 "더 알아보기" 클릭 시)**:
> 구체적 BCS 수치 + 위험 설명 + 학술 근거

받아들일 준비가 됐을 때만 구체 정보 노출.

### 의료 사안 정책
- 시스템은 의료 판단 하지 않음
- "이런 신호가 있어요. 수의사 상담을 권해드려요" 로만 표현
- 시스템의 한계를 정직하게 전달

---

## 5. 능동 개입 (push / 알림) 정책

### 빈도 상한
- **사용자당 주당 최대 2건** (1건 정기 체크인 + 1건 능동 개입)
- 의료 위험·결제 실패 같은 critical 은 별도

### 거부 옵션 항상 제공
- 모든 개입 메시지에 **"지금은 괜찮아요"** 버튼
- 거부해도 시스템 계속 작동 (다음 추천에 학습 반영)
- 거부 빈도 높으면 시스템이 자동으로 개입 빈도 ↓ (negative feedback)

### 견주 자율성 우선
- 견주가 "나는 계량컵 그대로 쓸래" 라고 명시하면 시스템이 더 이상 저울 권유 안 함
- 그 결과 신뢰도가 낮아지는 건 그대로 반영
- **강요하지 않는다** 가 핵심 원칙

---

## 6. 첫 4주의 보호

D2C 정기구독 첫 4주 이탈률이 전체 60%. 이 기간엔 시스템 정확도보다 사용자 안전감 우선.

| 주차 | 시스템 행동 |
|------|------------|
| 1주차 | **아무것도 요구하지 않는다**. 가입 → 기초 정보 5개 → 추천 → 사료 배송. 신뢰도 점수 미표시. |
| 2주차 | "어땠어요?" 1분 짜리 간단 체크 1회 |
| 3주차 | "더 정확하게 케어하고 싶다면..." 옵션 제시 (강제 X) |
| 4주차 | 첫 신뢰도 점수 표시 (이때 이미 충분히 높음). 첫 추천 조정 |

### 첫 추천의 의도적 보수성
- 4주차 첫 조정은 **무조건 안전한 방향** (예: -5% 권장량 감소)
- 사용자가 "효과 있네" 경험하면 신뢰 정착
- 4주 이후에야 정밀 조정 시작

---

## 7. "모름" 옵션 항상

모든 메타데이터 입력 필드에 **"모름 / 기억 안 남"** 옵션 제공.

- 입력 안 하면 신뢰도가 낮아지지만 시스템은 계속 작동
- 견주에게 "모름이라고 답해도 괜찮다"는 심리적 안전감
- 강제 입력 → 거짓 입력의 악순환 차단

---

## 8. 추천 근거 항상 노출

모든 추천에 **"왜 이렇게 추천했나요?"** expandable 버튼.

```
권장 칼로리: 380~420 kcal/일

[왜 이렇게?] ▼
  - 초롱이는 5세 골든리트리버, 중성화, 활동 보통
  - NRC RER = 70 × 28kg^0.75 = 약 850kcal
  - 중성화 계수 1.0 × 활동 계수 1.2
  - 현재 측정 메타데이터 기준 맞춤도 78%
```

블랙박스 거부감 해소. 자연어 + 수치 병기.

---

## 9. 견주 자율 수정 권한

견주가 직접 시스템을 수정할 수 있어야 함 (User Sovereignty).

- "초롱이 추천보다 활동량 더 많아요" → 활동량 신뢰도 가중치 슬라이더 조정
- "이 사료보다 다른 라인 선호" → 사료 추천 비율 직접 조정
- 시스템이 일방적으로 결정하지 않음

---

## 10. 게이미피케이션 보상 정책

### 즉시 보상 (action → reward 짧은 사이클)
- 측정 도구 개선 시 → **즉시 1,000원 적립금**
- 추상적 점수가 아닌 실제 화폐 가치

### 연속 체크인 스트릭
| 주 | 보상 |
|----|------|
| 4주 연속 | 무료 간식 1봉 |
| 12주 연속 | 신제품 무료 샘플 |
| 24주 연속 | "정밀 케어 마스터" 뱃지 + 10% 영구 할인 |

### 마일스톤 축하
- "초롱이의 정밀 케어 100일"
- "초롱이가 이상 체중에 도달했어요!"
- 견의 성취를 축하 (시스템 성공 아님)

---

## 11. 사진 촬영 정책

### 사진은 진짜 옵션
- 메인 플로우에서 사진 요구 완전 제거
- "월 1회 사진 1장 올리면 30% 할인 쿠폰" 같은 **큰 보상으로만 유도**
- 보상이 노력 대비 명확히 클 때만 사용자가 움직임

### "1장만 좋아도 OK"
- 측면 + 위 두 장 다 요구 X
- 둘 중 하나만 있어도 작동 (신뢰도는 더 낮음)

### 친구가 찍어주기
- 견주 혼자 측면 사진 찍기 어려움 (견을 잡으면서)
- "측면 사진 한 장 부탁" 공유 링크 전송 기능
- 친구가 앱 설치 없이 웹에서 사진 업로드

---

## 12. 감정 디자인 — 견 중심

- 메인 화면 상단은 항상 **견의 이름과 사진**
- "초롱이의 오늘" 같은 감정적 헤더 (데이터·점수는 그 아래)
- 12주 시계열 그래프 + "초롱이가 4주 전보다 더 건강해졌어요" 자연어 요약
- 견주가 "내가 잘 키우고 있구나" 느끼게 함

---

## 13. 페르소나별 분기

견주는 한 종류가 아님. 클러스터별로 UX 분기:

| 페르소나 | 톤 / UI |
|----------|---------|
| **데이터 신뢰 타입** | 풍부한 차트·수치·그래프 / 신뢰도 항상 표시 / 근거 상세 |
| **감성 케어 타입** | 견 사진·이름 강조 / 수치 최소 / 감정 메시지 |
| **편의 우선 타입** | 메인 플로우 단순 / 알림 최소 / 자동 입력 인프라 최대 |
| **수의사 의존 타입** | 동물병원 연동 강조 / 수의사 추천 신뢰 / 입력 부담 최소 |

신규 사용자는 입력 패턴 (도구 선택, 첨부, 빈도) 으로 초기 클러스터 추정.

---

## 14. 사업 모델 차원의 UX

### 구독 유연성 (이미 적용)
- 일시 중지 / 건너뛰기 / 수량 조절 무조건 쉽게
- 해지 페이지에 만류 문구 금지

### 가족 다중 계정 (추후)
- 한 가구 여러 사람이 같은 견 케어
- 가족 구성원 모두 같은 데이터 가시
- 한 가구에 여러 마리 자연스럽게 지원

---

## 적용 우선순위

이 가이드라인은 새 기능 개발 시 모두 따른다. 기존 코드 sweep 은 phase 별로 진행:

1. **Phase D1** (이 문서 + 즉시 적용): voice 정책 명문화, NextActionCard dismiss
2. **Phase D2**: 첫 4주 보호 + 부정 다단계 + 사료 배송 무게 추적
3. **Phase D3+**: 점진적 입력 / 친구 link / 가족 계정 / 페르소나 분기

PR / commit 시 "voice-guidelines 위반 여부" 자체 점검.
