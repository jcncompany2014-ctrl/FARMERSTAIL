# 급여량 알고리즘 — 사장님용 설명서

> **이 문서는 코드를 모르고도 읽을 수 있게 쓴 것입니다.**
> "우리 고객의 밥 양이 어떻게 정해지는가"를 처음부터 끝까지 풀어 씁니다.
> 마지막에 **앞으로 뭘 해야 하는지**가 있습니다.
>
> 최종 갱신: 2026-07-16

---

## 0. 한 문장으로

**강아지 체중으로 기초 소모 열량을 구하고 → 그 아이 사정에 따라 계수를 더하고 빼서 하루 필요 열량을 정한 다음 → 레시피 칼로리로 나눠 그램을 냅니다.**

```
   체중  →  RER(가만히 있어도 쓰는 열량)
                    ↓  × 계수(활동·나이·체형·중성화…)
              MER(하루 필요 열량)
                    ↓  ÷ 레시피 kcal/100g
              하루 몇 그램
```

---

## 1. 왜 이렇게 하나 — 원칙 세 개

### ① 숫자는 AI에게 맡기지 않는다

급여량은 **살아 있는 동물이 먹을 양**입니다. AI가 그럴듯하게 지어내면 안 되고, 같은 아이를 두 번 계산하면 같은 값이 나와야 하고, 왜 그 숫자인지 설명할 수 있어야 합니다.

그래서 **숫자는 전부 규칙(순수 함수 + 테스트)** 으로 냅니다. AI는 **규칙으로 못 푸는 것**에만 씁니다 — 그 아이의 사료 브랜드, 만성질환, 복용 약, 변 상태를 읽고 **말로 설명을 붙이는 일**.

이렇게 갈라두면 AI가 실패하거나 비용 한도에 걸려도 **급여량은 멀쩡합니다.**

### ② 곱하지 않고 더한다 (감산 지배형)

예전엔 이렇게 했습니다: `활동계수 1.2 × 체형계수 0.85 × 중성화 0.9`

**문제:** 곱셈은 겹칩니다. "저활동"에 이미 '적게 움직인다'가 들어 있는데 거기 또 중성화 0.9를 곱하면 **같은 이유로 두 번 깎입니다.** 그리고 곱셈은 값이 순식간에 튑니다.

**지금:** 기준값 하나를 깔고 **더하기/빼기만** 합니다.

```
기본 1.4  (중성화 · 실내 · 저활동 = 한국에서 제일 흔한 강아지)
  − 0.2   10세 이상
  − 0.1   7~9세
  − 0.1   쉽게 찌는 체질 또는 비만 경향 견종
  − 0.1   거의 안 움직임
  + 0.2   중성화 안 함
  + 0.2   저체중 (BCS 3 이하)
  + 0.1   활발함 (보호자가 그렇다고 말함)
  + 0.2~0.4  활발함 (측정 기록 같은 증거가 있음)
  + 0.15  진짜 실외 거주 + 추운 환경
  ────────────────────────────
  결과는 1.0 ~ 2.0 사이로 자름
```

**핵심은 "기본 1.4를 낮게 깔았다"** 입니다. 대부분의 한국 강아지는 중성화했고, 실내에 살고, 많이 안 움직입니다. 그러니 **기본을 그 상태로 잡고 대부분 빼는 방향**으로 갑니다. 더하는 건 **증거가 있을 때만** — 그래서 "활발해요"라고 말만 하면 +0.1이지만, 측정 기록이 있으면 +0.2~0.4입니다.

> **왜 낮게 까나:** 과식은 되돌리기 어렵고(비만), 부족은 금방 눈에 보입니다(살 빠짐). 틀릴 거면 **적게 주는 쪽으로 틀리는 게** 안전합니다.

### ③ 이중차감 금지

"비만 경향 견종(래브라도 등)"과 "설문에서 쉽게 찐다고 답함"이 **둘 다 걸려도 −0.1은 한 번만** 뺍니다. 같은 사실을 두 경로로 알았을 뿐이니까요.

## 2. 실제 계산 순서

### 1단계 — 체중 정리

- 0.5kg ~ 100kg 밖의 값은 잘라냅니다 (0kg이 들어오면 계산이 통째로 망가집니다)
- **측정 신뢰도가 낮으면 안전한 쪽으로 살짝 당깁니다.** 예를 들어 비만 관리 중인데 체중을 대충 재셨다면, 계산 체중을 조금 보수적으로 잡습니다. (신뢰도가 높거나 대칭적인 목표면 건드리지 않습니다)

### 2단계 — RER (가만히 있어도 쓰는 열량)

```
RER = 70 × (체중kg)^0.75
```

지수식입니다. 2kg 이하 초소형견은 이 식이 부정확해서 별도 보정이 들어갑니다.

### 3단계 — 계수 정하기 (여기서 갈래가 셋)

**갈래 A · 자견(성장기)**

성견 예상 체중을 아는 경우 — NRC 2006 정확식:
```
필요열량 = 130 × 체중^0.75 × 3.2 × (e^(−0.87 × 성장률) − 0.1)
```
> ⚠️ 앞 상수가 **130**입니다. 예전 스펙에 70으로 적혀 있었는데 그러면 **약 46% 적게** 줍니다. 성장기 강아지에게 밥을 절반만 주는 셈이라 절대 바뀌면 안 됩니다.
>
> 토이 견종은 이 식이 과대추정해서 **15% 낮춥니다.**

예상 체중을 모르면 간이 근사: 4개월 미만 ×3.0 / 8개월 미만 ×2.5 / 그 외 ×2.0

**갈래 B · 과체중 (BCS 6 이상)**

**현재 체중이 아니라 이상 체중으로 RER을 다시 계산**하고 계수는 1.0에서 시작합니다.

> 이게 중요합니다. 뚱뚱한 아이에게 **지금 체중 기준으로 밥을 주면 지금 체중이 유지됩니다.** 이상 체중 기준으로 줘야 그 체중으로 갑니다.
>
> 하한은 이상체중 RER × 0.8 — 이 밑으로는 안 내려갑니다(굶기면 근손실).

**갈래 C · 그 외 성견/노령견**

위 1절의 **계수 사다리**를 탑니다.

### 4단계 — 임신 · 수유

일반 계수를 **쓰지 않고 교체**합니다(NRC 배수). 임신·수유 중엔 영양의 대부분이 태아·젖으로 가서 평소 유지 열량 계산이 의미가 없습니다. **수의 상담 배너가 강제로 뜹니다.**

> 사장님 확정(2026-07-12): 스펙엔 "계산 중단하고 수의사에게 보내기"였는데, 그러면 **구독 흐름이 끊기고** "긍정 먼저" 원칙과 부딪힙니다. 그래서 **계산은 해주되 배너를 강제 노출**하는 절충으로 갔습니다.

### 5단계 — 안전 상한/하한

계수가 5.0을 넘거나 0.5 밑으로 가면 **입력 오류로 보고 잘라냅니다.** (NRC 기준 가장 극단적인 경우 — 새끼 8마리 수유 중인 저체중 엄마 — 도 5.0 이하입니다)

### 6단계 — 그램으로

```
하루 필요 열량(MER) = RER × 계수
하루 그램 = MER ÷ (레시피 kcal/100g) × 100
```

레시피 칼로리 (2026-07-11 검정 확정):

| 레시피 | kcal/100g |
|---|---|
| 치킨 | 115 |
| 오리 | 120 |
| 흑돼지 | 115 |
| 한우 | 120 |

### 7단계 — 화식 비율

고객이 고른 티어만큼만 화식으로 채웁니다: **곁들임 30% / 반반 60% / 완전 100%.** 나머지는 기존 사료입니다.

> ⚠️ **지금 우리는 "그럼 사료는 몇 g 줄이세요"를 안 알려줍니다.**
> 분석 화면은 화식 g만 비율만큼 줄여서 보여주고 끝납니다. 보호자 입장에선
> **"화식 108g"만 듣고 사료는 원래대로 주기 쉽습니다 → 과식.**
> 계산기(`lib/feeding-plan.ts`)는 이미 만들어져 있는데 그걸 부르는 곳이
> 익명 설문 하나뿐이고, 거기선 **화식 100% 기준으로만** 불러서 사료 g이 늘 0입니다.
> → [업그레이드 목록](./ALGORITHM_UPGRADES.md) 3번.

### 8단계 — 간식 차감

간식을 준다고 하면 **그만큼 밥을 줄입니다.** AAFCO/WSAVA "10% 룰" — 간식은 하루 열량의 10% 이내여야 하고, **간식 위에 밥을 그대로 주면 과식**입니다.

---

## 3. 결과가 얼마나 정확한가 — 정직하게

**±30%입니다.** 이건 우리가 못해서가 아니라 **개체차가 원래 그렇습니다.** 같은 5kg 강아지 둘도 실제 소모 열량이 30% 넘게 차이날 수 있습니다.

그래서 **진짜 정확도는 계산이 아니라 재측정에서 옵니다.** 2~4주 뒤 체중을 다시 재서 "예상보다 덜 빠졌네 → 계수를 낮춘다"로 **그 아이에게 맞춰 수렴**시킵니다. 이게 우리가 앱을 만든 이유이기도 합니다.

## 4. 지금 코드가 어디에 있나

| 무엇 | 파일 |

| **메인 계산 (실제로 도는 것)** | `lib/nutrition.ts` — `calculateNutrition()` |
| 계수 사다리 | `lib/calorie-v2/adapter.ts` → `lib/calorie-v2/engine.ts` 의 `calculateAdultFactor` |
| 숫자 상수 (BASE 1.4, 각 델타) | `lib/calorie-v2/constants.ts` |

| 전환 급여표 (첫 2주) | `lib/transition-plan.ts` |

### ⚠️ 헷갈리기 쉬운 것

**`lib/calorie-v2/engine.ts`에는 "완성된 새 계산기"(`computeFeedingPlanV2`)가 통째로 들어 있는데 — 프로덕션은 그걸 안 씁니다.**

거기서 **계수 사다리 하나만 뽑아 `nutrition.ts`에 이식**했고, 나머지(기초대사량·간식 감산·성장/감량 분기·건사료 칼로리)는 `nutrition.ts`가 자기 방식으로 갖고 있습니다.

**즉 engine.ts의 상위 파이프라인은 테스트만 통과하는 상태입니다.** 그 안에는 **이미 뒤집힌 결정**(임신/수유 = 계산 중단)도 들어 있으니 **그 코드를 프로덕션 동작의 근거로 읽으면 안 됩니다.** 정본은 언제나 `lib/nutrition.ts`입니다.

→ 이걸 어떻게 할지는 **5절**에 있습니다.

## 5. 앞으로 뭘 해야 하나

### 지금 당장

**① `engine.ts`의 안 쓰는 부분을 어떻게 할까**

세 가지 길이 있습니다:

- **(a) 지운다** — 지금 계산이 잘 돌고 있고, 안 쓰는 코드가 있으면 나중에 그걸 읽고 오해합니다. 필요하면 git에서 꺼냅니다.
- **(b) 냅둔다** — 언젠가 갈아탈 거니까. 대신 지금처럼 "미연결"이라고 크게 붙여둡니다.
- **(c) ★사장님 선택 — "업그레이드 대기 목록"으로 따로 보존한다.** engine.ts에 있는 것 중 **지금 nutrition.ts에 없는 좋은 것**만 골라 [ALGORITHM_UPGRADES.md](./ALGORITHM_UPGRADES.md)에 "이건 이래서 좋고, 붙이려면 이걸 해야 한다"로 적어뒀습니다. **아이디어는 남고, 나중에 부품만 하나씩 옮기면 됩니다.**

### 데이터가 쌓이면 (고객 생긴 뒤)

**② 기본값 1.4를 다시 본다**

지금 1.4는 **추정치**입니다(AAHA 중성화 성견 1.4~1.6의 하단과 비활동/비만경향 1.0~1.2의 상단이 만나는 지점). 실제 고객 재측정 데이터가 쌓이면 **1.3으로 낮추는 게 맞을 수 있습니다.** 우리 고객이 대부분 실내 소형견이면 1.4도 여전히 후할 수 있거든요.

> 판단 기준: 재측정에서 **"예상보다 살이 찐 아이"가 "빠진 아이"보다 확연히 많으면** → 1.4가 높다는 뜻.

**③ 사료 데이터베이스 채우기**

`kibble_products` 테이블은 **이미 만들어져 있고 컬럼도 완벽합니다**(칼로리·성분 5종·출처·검색어). 그런데 **0행**입니다. 즉 **코드 문제가 아니라 데이터가 없는 것**입니다.

한편 설문은 **이미** 고객에게 사료 라벨 kcal을 묻고, 모르면 그 브랜드를 `kibble_requests`에 쌓습니다(= 다음 매장 투어 조사 목록). **자가성장 장치는 돌고 있습니다.**

> ⚠️ 다만 **그렇게 받은 라벨 kcal이 지금 어떤 계산에도 안 들어갑니다.** → 업그레이드 목록 1번.

**④ 각 델타(−0.1, +0.2…)도 실측으로 재튜닝**

이 숫자들도 문헌 기반 추정입니다. 재측정 데이터가 답을 알려줍니다.

### 하면 안 되는 것

- **성장기 앞 상수 130을 70으로 바꾸지 말 것** (46% 과소급여)
- **계수를 곱셈으로 되돌리지 말 것** (이중차감)
- **레시피 칼로리는 `skuModel.ts`의 값만 정본** — 주석이나 문서에 적힌 옛 설계값(닭130·오리150…)을 믿지 말 것
- **급여량 숫자를 AI에게 시키지 말 것**

## 6. 용어 사전

| 말 | 뜻 |
|---|---|
| **RER** | 가만히 있어도 쓰는 열량 (Resting Energy Requirement) |
| **MER** | 하루 실제 필요 열량 (RER × 계수) |
| **BCS** | 체형 점수 1~9. 5가 이상적, 6 이상 과체중, 3 이하 저체중 |
| **IBW** | 이상 체중 (Ideal Body Weight) |
| **계수(factor)** | RER에 곱해서 MER을 만드는 배수 |
| **감산 지배형** | 기본을 낮게 깔고 대부분 빼는 방식 |
| **easy keeper** | 적게 먹어도 살찌는 체질 |
| **NRC / AAFCO / FEDIAF** | 반려동물 영양 기준을 내는 기관들 |
