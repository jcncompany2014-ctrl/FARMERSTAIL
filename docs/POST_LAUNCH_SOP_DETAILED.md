## 🌅 D+1 ~ D+7 — Daily Routine (15분 × 7일)

기존 `POST_LAUNCH_WEEK1.md` 의 `Day 1-7 매일 아침 루틴` 참조. 여기서는 **의사결정 트리** 만 보강.

### 매일 아침 KPI 보고 결정해야 하는 것

```
┌─ 어제 가입 = 0?
│  ├─ Yes → CTA 메시지 톤 약함 → 내일 인스타 스토리에 강한 hook 1개
│  └─ No → 가입 → 결제 funnel 확인
│
├─ 가입 후 결제 전환 = ?%
│  ├─ < 10% → 가격 페이지 / 분석 흐름 어디서 막힘 (Sentry "abandoned cart" 이벤트 확인)
│  ├─ 10-25% → 평균 — 유지
│  └─ > 25% → 매우 좋음 — 광고 ROI 검토 시점
│
├─ Sentry 새 error = ?
│  ├─ 0 → 다음
│  ├─ 1-3 → 종류별 분류, P0 (결제/auth) 즉시 fix, 나머지 backlog
│  └─ 4+ → 즉시 stack trace 확인, 공통 root cause 추정
│
├─ 카트 담기 → 결제 = ?%
│  ├─ < 30% → 결제 폼 / 배송 정보 입력 stuck (히트맵 분석)
│  ├─ 30-50% → 평균
│  └─ > 50% → 매우 좋음
│
└─ 정기구독 신청 = ?
   ├─ 0건 → 정기구독 hook 약함 (CheckoutForm 정기 가입 incentive 검토)
   ├─ 1-2건 → 평균 시작 — 유지
   └─ 3+건 → 정기구독 효과 — 다음주 광고 keyword "정기구독" 강화
```

### 매일 18시 — 다음날 콘텐츠 결정 (5분)

| 오늘 트래픽 패턴 | 내일 발행 콘텐츠 |
|---|---|
| 인스타 좋아요 > 100 | 같은 톤 유지, 후속 캐러셀 |
| 인스타 좋아요 < 30 | 톤 바꿈 — 보호자 인터뷰 / 강아지 영상 |
| 가입 5+명 | 후기 콘텐츠 (가입자에게 후기 요청) |
| 결제 3+건 | 언박싱 사진 / 영상 (구매자 동의 후) |
| 환불 1+건 | 다음날 발행 X — 침묵, 환불 처리 우선 |

## 🚨 위급 상황 매트릭스 (상세)

### 시나리오 A: 사이트 다운 (HTTP 500 폭주)

**증상**: Vercel logs 500 빈도 ↑↑, Sentry "Internal Server Error" 5+ 건

**즉시 대응 (5분 안)**:
1. Vercel → Deployments → 어제 Production 으로 **즉시 롤백**
2. Sentry → 최신 issue → stack trace 확인
3. 본인 노트에 "yyyy-mm-dd hh:mm — 사이트 다운 → 롤백, 원인: ___"

**다음 단계 (30분 안)**:
1. 로컬에서 동일 재현 시도
2. 원인 fix → npm run verify → push
3. Vercel 재배포 → 정상 확인 → 인스타 스토리 "잠시 점검 있었어요 [복구 이모지]"

### 시나리오 B: Toss 결제 30%+ 실패

**증상**: payment_events 30분 내 5건 + 실패 비율 30%↑

**즉시 대응 (10분 안)**:
1. Toss 가맹점 콘솔 → 가맹점 상태 (정상 / 점검 / 정지) 확인
2. 점검 중이면 → 인스타 스토리 "Toss 점검 중 — 1시간 후 재시도 안내"
3. 가맹점 정지 → 즉시 Toss 1:1 문의 (전화 1577-7779)

**다음 단계 (1시간 안)**:
1. 영향 받은 사용자 메일 발송 — "결제 실패, 재시도 가능 시점 안내"
2. 재시도 시점 SNS 공지

### 시나리오 C: 알레르기 클레임 (펫푸드 최우선)

**증상**: 사용자가 "우리 강아지가 사료 먹고 토했어요 / 가려워해요"

**즉시 대응 (30분 안)**:
1. **사진 + 시간 + 증상 모두 요청** (메일/카카오 가장 빠른 채널)
2. **전액 환불 + 사과** (조건 없이)
3. 동물병원 진료비 영수증 제출 시 **환급 약속**
4. 해당 SKU 알레르기 표시 점검 — DB `products.allergens` 확인

**다음 단계 (2시간 안)**:
1. 같은 단백질 라인 다른 주문자 (지난 7일) 명단 추출
2. 사전 알림 메일 "혹시 같은 증상 있으셨나요?" (조심스럽게)
3. 만약 같은 보고 2건+ → 해당 SKU 즉시 판매 중지 + 전수 환불

**법적**:
- 1건은 individual case — 처리 + 보상으로 종결
- 2건+ → 사료관리법 §17 (불량 사료 회수) 검토 → 변호사 자문

### 시나리오 D: 부정 환불 의심

**증상**: 7일 내 같은 IP/이메일/배송지에서 환불 3건+

**증거 수집 (15분 안)**:
1. `/admin/orders` 동일 IP/배송지 검색
2. `payment_events` 패턴 분석 — 환불 시점, 사유
3. `auth.users` 가입 IP / 디바이스 검색

**조치**:
- 3건 패턴 → 사용자 일시 정지 (DB `profiles.status='suspended'`)
- 메일 발송 "이용약관 위반 의심 — 소명 요청"
- 응답 없거나 부족 → 사용자 영구 정지 + 카드사 분쟁 등록

### 시나리오 E: chargeback (카드사 분쟁)

**증상**: Toss 가맹점 콘솔 → 분쟁 알림 1건

**즉시 대응 (1시간 안)**:
1. Toss 콘솔 → 분쟁 상세 → 사유 확인 (단순 / 사기 / 미수령)
2. 우리 측 증거 수집:
   - 결제 로그 (payment_events)
   - 배송 송장 (실 수령 사진)
   - 사용자 가입 정보 (IP, 디바이스)
3. Toss 콘솔에 증거 업로드 + 소명서 작성

**예방**:
- chargeback 월 3건+ → Toss 가맹점 위험도 ↑
- 5건+ → 가맹점 정지 위험
- 즉시 보안 조치: 결제 추가 인증 (3D Secure) 의무화 검토
