## 1. 마케팅 방향  _(회차92 — 코드 기반 1차 작성)_

### 1-A. 진짜 차별점 (코드에서 확인된 사실 → 마케팅 자산)
점검하며 본 파머스테일의 **실제 해자**는 "신선함"이 아니라 아래 3가지입니다. 한국 신선펫푸드 다수가 "HACCP·신선·휴먼그레이드"로 경쟁하는데, 그건 이미 레드오션 카피라 차별화가 약합니다. FT가 코드 수준에서 실제로 가진 건:
1. **임상 영양 과학의 깊이** — NRC2006·AAFCO2024·WSAVA 기준, RER/MER·BCS9점·MCS·Bristol·Ca:P비·CKD IRIS 병기·췌장염 중증도 게이트까지. 경쟁사가 "좋은 재료"를 말할 때 FT는 "수의영양학 그대로 계산"을 보여줄 수 있음.
2. **정직성(표시광고 규율)을 마케팅으로** — 질병 단정 안 함, "효능 단정하지 않을게요"(why-fresh), 자가보고 vs 수의사확진 라벨, 체중 측정도구 신뢰도 표시. **과장이 만연한 카테고리에서 "과장 안 하는 브랜드"가 곧 신뢰 포지셔닝**. 정직이 비용이 아니라 차별점.
3. **개인화 피드백 루프(특허)** — 설문→맞춤설계→2/4주 체크인→알고리즘 재조정. "점점 똑똑해지는" 경험은 1회성 신선식이 못 따라오는 리텐션 해자.
+보조: **vet-share**(수의사용 read-only 공유, 신뢰도 라벨 포함) = "수의사와 협업하는 브랜드" 포지션(반-수의사 아님).

### 1-F. 구독 전용 전환의 포지셔닝 함의 _(회차315 — 2026-06-26 피벗 기반·구독전환 작업 관찰)_
사장님이 낱개 커머스·무료배송·위시리스트를 전부 폐지하고 **구독 전용**으로 전환(2026-06-26). 단순 기능 축소가 아니라 **포지셔닝 강화 기회**:
1. **메시지 단순화 = 전환 자산**. 낱개+구독 병행은 "뭘 살까" 결정 부담을 줬는데, 구독 전용은 모든 CTA가 설문 퍼널(`/start`) 하나로 수렴(코드상 ~20 진입점 정합 확인) → "우리 아이 맞춤 식단 시작" 단일 행동. 결정 마찰↓·퍼널 누수↓.
2. **무료배송 메시징 재설계 = 가장 시급(#86)**. 현재 웹 프로모 "3만원 이상 무료배송" + 구독 가입가 배송비 임계(3만원)가 잔존 → 구독 모델에 부적합·메시지 분열. **구독은 배송비를 구독가에 번들("배송비 걱정 없는 정기배송")이 깔끔**. "임계 채우러 더 담기" 넛지는 낱개식 사고라 구독과 안 맞음. 배송비 정책 단일화가 결제·표시광고·전환 메시지를 동시에 정리.
3. **관계형 브랜드로 이동**. 낱개=거래, 구독=관계. 1-E 리텐션 자산(체크인 사이클·streak·캐릭터 농장·개인화 루프)이 이제 **부가가 아니라 핵심 가치제안** — "한 번 사는 사료"가 아니라 "아이와 함께 진화하는 식단 파트너". LTV 중심(CAC 회수·해지 방어)이 마케팅 KPI의 중심으로.
4. **리스크: 진입장벽↑**. 구독 전용은 "한 번만 써볼게" 고객을 잃음 → **체험팩/첫 박스 할인**(이미 "첫 주문 50%" 자산 보유)이 약정 부담을 낮추는 다리. 익명 설문→결과 직전 가입(이미 구현)이 약정 전 가치를 먼저 보여주는 좋은 설계.
**→ 종합**: 구독 전용은 메시지·퍼널·리텐션을 단순·강화하는 방향. 단 ①무료배송 메시징 정리(#86) ②체험팩 진입 다리 강화가 전환 보호의 양대 선결. _(코드 변경 아님·전략 관찰)_

### 1-G. 구독 전용 성장 레버 (P5 발굴·회차319~322·코드 기반)
인프라(라이프사이클 5단계·churn 방어)는 갖춰졌으니 이제 **'성장'** — 매출·리텐션을 키울 코드 기반 기회:
1. **구독 add-on (객단가/AOV) — 미구현·ROI 최고**: 구독 박스에 보조제/토퍼를 add-on으로 붙이는 흐름이 **없음**(`subscribe/*` 업셀 0). 추천 엔진은 이미 미판매 보조제를 추천(2-A=검증된 수요)인데 **살 경로가 없어 수요가 새는 중**. 구독 박스 add-on = 정석 AOV·리텐션 레버(회당 매출↑·인프라 위 얹기라 재구축 아님).
2. **다견 객단가 — asset 이미 있음**: 구독은 **강아지별**(`subscriptions.dog_id`)이라 2마리=2구독=객단가 2배가 **코드로 지원됨**. 마케팅 각도 "여러 아이도 각각 맞춤"(per-dog 개인화). 단 **둘째 아이 등록 인센티브**(다견 할인/묶음)는 미구현 → 다견 획득 캠페인이 미개봉 기회.
3. **질림 리텐션 (rotation 가시화)**: 박스는 알고리즘(2종)+`protein-rotation` 자동변주라 수동 변경 없음(설계·결정피로↓). 질림 방어 강화 옵션 = **다음 박스 미리보기**(기대감·rotation 가시화) — 자동화 철학과 균형. 반복 박스 churn 데이터로 필요성 판단(지금 단정 X).

4. **referral 바이럴 (CAC) — 구독-네이티브 보상 기회**: #70 milestone(친구 5/10/20명) 작동 중이나 보상이 P 적립. 구독 전용 모델엔 **"친구 N명 → 1개월(=구독 1회) 무료"**가 P 적립보다 강력 — 피초대자도 구독으로 유입(CAC↓), 초대자는 약정 부담↓. 현 UI "1개월 무료" 표시 ↔ 실제 P 적립 불일치(#70) 정합 시 **구독-네이티브 보상으로 통일** 검토. 솔로 D2C의 CAC 핵심 무기.

**→ 성장 우선순위: ① add-on(AOV·ROI 최고) > ② referral 보상 구독화(CAC) > ③ 다견 인센티브 > ④ rotation 가시화.** 전부 기존 인프라 위 '얹기'라 큰 재구축 불필요. _(단 #86 배송비·#9 streak 등 PG前 선결이 성장보다 먼저.)_

## 2. 상품 로드맵 제안  _(회차93 — 코드 기반 1차 작성)_

> 근거: `lib/personalization/skuMap.ts`·`lines.ts`·survey·analysis 추천 로직에서 확인된 사실.
>
> **핵심 렌즈(회차153)**: 앱은 이미 *미판매* 제품(연어·피쉬오일/글루코사민/프로바이오틱스 등 보조제·토퍼)을 **추천**하고 있다 = 검증된 미충족 수요. 즉 **추천 엔진이 곧 상품 로드맵 우선순위기** — 추천 빈도(어느 단백/보조제/condition가 얼마나 자주 추천되나)를 집계하면 출시 순서를 *데이터로* 정렬할 수 있다. **#56**(챗·copy가 미출시 "연어 화식 FT-S03"을 이름으로 추천 = 지금 새고 있는 수요)이 그 증거. 출시 전까지는 추천을 막거나 "곧 출시" 정직 표기로 신뢰 보호.

### 2-A. 연어 라인 출시 = 1순위 (코드가 이미 전제)
skuMap.ts에서 **연어 라인은 현재 "미출시/준비중"이라 오리로 자동 대체** + 사용자에게 정직하게 안내("연어 준비중→오리 대체, 오메가3는 연어보다 낮아 피쉬오일 보조 권장"). 즉:
- 시스템이 이미 연어 출시를 전제로 설계됨(출시 시 자동 반영). **피부·인지(오메가3/DHA) 케어 수요를 지금 못 받고 있음** = 명확한 미충족 수요.
- 정직한 미출시 표기는 신뢰 자산이지만, 길어지면 "5종"이라 광고하며 4종만 파는 셈 → 출시 가속 또는 마케팅에서 "현재 4라인+연어 곧" 정직 표기 권장.

### 2-B. 보조제(영양제)를 자사 SKU로 — 추천을 매출로
분석/설문/vet-share가 **보조제를 추천**(getSupplements/getConditionSupplements: 피쉬오일·글루코사민·프로바이오틱스 등)하는데, 코드상 보조제가 **자사 판매 SKU로 보이지 않음**. 추천만 하면 **고객이 그 보조제를 타사(아마존·쿠팡)에서 삼 = 매출·신뢰·데이터 누수**.
- 추천 보조제를 자사 SKU로 → 정기배송에 add-on(AOV↑) + "우리가 추천한 걸 우리가 책임지고 공급" 신뢰 루프 완결.
- **연어 미출시 기간 피쉬오일(EPA/DHA)은 즉시 수요** — 코드가 이미 그 보조를 권하므로 첫 add-on SKU로 자연스러움.
- 단 보조제도 표시광고 규율(효능 단정 금지) 유지 — FT 정직성 톤과 일관.

### 2-C. 간식/토퍼 확장
설문이 `snackFreq`(간식 빈도)를 묻고 토퍼(야채/육류) 개념이 이미 있음 → **같은 5단백 단일재료 간식 라인**(건조/동결건조)이 자연 확장. 임펄스·AOV·"주식과 같은 단백 = 알레르기 안전" 메시지. 토퍼도 단백/condition별 확장 여지.

### 2-D. 가격/번들 전략 (budget tier 활용)
설문에서 `budget_tier` 4단계를 받고(미응답 시 mix50 default), 분석 직전 budget step으로 가격 앵커("jab")를 형성. 이 응답 데이터를 활용해:
- 예산대별 박스 구성(단백 비율·토퍼·용량) 자동 추천 + 체험팩→정기 전환 앵커 정교화.
- add-on(보조제·간식) 번들로 객단가 상향.

### 2-E. 처방식/condition 라인은 후순위·신중
`chronic-sku-mapper`가 신장 처방식을 **"본 라인업 외 + 수의사 상담 강권"**으로 이미 정직하게 선 그음. 치료식(renal 등)은 규제·수의·임상 장벽이 높음 → 직접 라인보다 **수의사 협업 채널 전용** 또는 장기 로드맵. 무리한 치료식 진입은 표시광고·신뢰 리스크.

### 4-B. 정직성을 명시적 신뢰 장치로

- 영양 근거 투명성: 분석에 이미 guideline_version·근거 표시 → "왜 이 수치인지" 더 펼쳐 보이면 임상 권위 강화.
- **표시광고 전수 audit 종합(#50~59, 회차125~141)** — 정직 규율은 *제품/분석 레이어*엔 견고(skuMap 미출시 대체·vet-report 출처라벨/면책·why-fresh "효능 단정 안함")하나 *마케팅 레이어*엔 과장 산재: 수의 자문/검수 클레임(#52·#54, **자문 수의사 실재 여부가 핵심 단일변수**)·"처방" 의료용어 편재(#53)·연어 미출시 노출(#56)·HACCP/농가수(30 vs 6)/원산지 미입증(#54·55·57). **역설**: 정직성이 진짜 차별점인데 일부 카피가 그걸 배신. → **사내에 이미 정답 톤이 있다**: `/why-fresh`(사실 비교 + "건강 효과 단정 안 할게요")와 `/vet-report`(자가측정·견주보고·알고리즘 출처 라벨 + 완전 면책)가 "과장 없이 신뢰 제시" 모범. 과장 페이지(science·our-food·about·partners·brand)를 **이 톤으로 통일**하면 법적 안전(PG·표시광고) + 포지셔닝 강화가 동시에. **잃을 게 없다 — 정직성이 곧 세일즈 포인트**. ### 4-C. 후기 전략 (가짜 없이)
