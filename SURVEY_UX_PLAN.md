# 설문 UX 개편 기획 — SURVEY_UX_PLAN

> 2026-07-12 사장님 지시: "설문 페이지 전체의 디자인이나 질문의 UI 구조" 업그레이드 기획.
> 사진·일러스트가 필요한 지점은 §4 에 구체 스펙으로 — 사장님이 준비.
> 전제: progressive disclosure 정돈(질환 게이트·산책 리드·근육 노령견만·식욕/만족도 삭제·
> GI/측정법 접기)은 이미 배포됨. 이 기획은 그 위에 얹는 **표현·구조 층**.

## 1. 현재 진단 (코드 전수 정독 기준)

### 1-1. 질문 UI 문법이 스텝마다 제각각 — 가장 큰 문제
같은 "하나 고르기"인데 스텝마다 다른 컴포넌트를 쓴다:

| 문법 | 쓰는 곳 | 생김새 |
|---|---|---|
| `s-chip` 칩 | 체중변화·간식·산책·주거·임신·모질 등 대부분 | 작은 알약 |
| `s-tile` 타일 | 주식 형태·화식 경험 | 아이콘+라벨+메타 3단 |
| `s-listbtn` 리스트 | MCS·Bristol·케어 목표 | 번호+아이콘+제목+설명+태그 |
| `s-seg` 세그먼트 | 알레르기 유무·질환 유무 | 2~3분할 바 |
| `s-card` 카드 | 예산 | 라디오형 큰 카드 |

사용자 입장에서 "왜 이 질문은 크고 저 질문은 작지?"가 매 스텝 리셋됨 →
학습이 안 쌓임. **정보의 무게와 UI 크기가 비례하지 않는 게 문제.**

### 1-2. 관찰형 질문이 텍스트 뿐 — 정확도·속도 다 손해
- 체형 3분해(갈비뼈·허리·배): "잘록하게 들어가요" 같은 말로만 판단. WSAVA BCS 는
  원래 **그림 차트**가 표준 — 보호자가 실루엣을 보고 "우리 애는 이거"를 고르는 방식.
- Bristol 변 7단계: 의학 표준 자체가 일러스트 차트. 현재는 lucide 원형 아이콘 7종으로
  추상화 — 정보가 사실상 없음.
- **그림이 들어가면 응답 속도·정확도가 다 오르고, 결과(BCS)가 곧 알고리즘 입력이라
  계산 정확도까지 직결.**

### 1-3. Diet 스텝이 여전히 과밀
정돈 후에도 주식→브랜드(+kcal)→간식(+kcal)→산책→(활동·격운동)→주거→화식경험 =
**한 화면에 최대 7~8 섹션 세로 나열**. 다른 스텝(체형 4문항, 변 1+1문항)과 밀도 차가
3배. 스크롤 압박이 이 스텝에 몰림.

### 1-4. 응답 피드백이 BCS 판정 카드 하나뿐
체형 3문항을 채우면 "BCS 5 이상적" 카드가 뜨는 건 좋은 패턴인데 **여기서 끝**.
나머지 스텝은 골라도 칩 색만 바뀜. "답할수록 분석이 만들어지고 있다"는 감각이 없음.

### 1-5. 스텝 전환이 즉시 교체 — 진행감 없음
goNext() 가 currentStep 만 바꿔서 화면이 뚝 바뀜. 전환 모션·완료 모먼트 없음.
8스텝(이제 7)을 "긴 폼"으로 느끼게 하는 요인.

## 2. 개편 방향 — 4개 트랙

### 트랙 A. 질문 문법 통일 (컴포넌트 그래머 확정)
정보 무게에 따라 3계층으로만:

1. **비주얼 초이스** (`s-visual-grid`): 그림 보고 고르는 관찰형 —
   체형 3분해, Bristol, 주식 형태. 2~4열 그리드, 일러스트+짧은 라벨.
   §4 일러스트 도착 전엔 현행 유지, 슬롯만 먼저 설계.
2. **칩** (`s-chip`): 한 단어 답 전부 — 빈도·유무·정도. (현행 유지, 최다)
3. **리스트 버튼** (`s-listbtn`): 설명이 필요한 선택 — 케어 목표, MCS, 예산.
   예산 s-card 는 s-listbtn 으로 흡수(한 문법 제거).

세그먼트(s-seg)는 "게이트 질문"(없어요/있어요) 전용으로 의미 고정.
→ **규칙이 3+1개로 줄면 사용자가 두 스텝 만에 학습 완료.**

### 트랙 B. 관찰형 질문 = 그림 선택 (§4 자산 필요)
- 체형 3분해: 텍스트 칩 → 실루엣 일러스트 그리드. 갈비뼈는 단면 触감 표현이
  어려우니 "옆구리를 만지는 손" 시퀀스 4컷.
- Bristol: 7단계 일러스트 차트 (밝고 무해한 톤 — 식품 브랜드라 톤 조절 중요.
  실사 금지, 부드러운 도형화).
- 주식 형태 3종: 그릇에 담긴 건식/화식/반반 일러스트 또는 실사진.
  (실사진이면 우리 화식 실물이 들어가 브랜드 자산 겸용 — 추천)

### 트랙 C. Diet 스텝 분리 — "식사" / "생활"
- **STEP 식사**: 주식(비주얼)→브랜드+kcal→간식+kcal→화식 경험
- **STEP 생활**: 산책(리드)→활동·격운동(조건부)→주거(+한랭)
분리 후 스텝 수는 +1 되지만 **스텝당 밀도가 균일**해져 체감은 짧아짐.
(진행바·STEPS 배열·validateStep 재배선, muscle 조건부와 같은 패턴)

### 트랙 D. 응답 피드백 + 전환 모먼트
1. **판정 카드 확장**: Bristol 고르면 "이상적인 변이에요 / 섬유를 보강할게요" 한 줄,
   간식 kcal 넣으면 "하루 권장량의 N%네요 — 밥에서 빼고 계산할게요" 한 줄.
   이미 계산 로직 있음(nutrition) — **미리보기만 노출**.
2. **스텝 전환**: 다음 누르면 완료 스텝이 살짝 접히며 좌로 슬라이드(180ms,
   reduced-motion 시 즉시). 마지막 스텝 → 분석 로딩은 현행 유지.
3. **스텝 인트로 통일**: 킥커(영문)+타이틀+서브 구조는 좋음 — 서브 카피만
   "왜 묻는지" 한 줄로 통일 (예: 변 스텝 "장 건강 신호예요 — 섬유 배합에 반영돼요").

## 3. 실행 순서 (5~15분 phase, 각각 독립 배포 가능)

| # | 작업 | 리스크 | 비고 |
|---|---|---|---|
| P1 | 그래머 통일 — 예산 s-card→s-listbtn, seg=게이트 전용 확정 | 낮음 | CSS/마크업만 |
| P2 | Diet 분리 (식사/생활 2스텝) | 중 | STEPS 재배선 |

| P6 | 카피 스윕 (서브 한 줄 통일) | 낮음 | |

P1~P4 는 자산 없이 지금 가능. P5 만 사장님 자산 대기.

## 4. 📸 사장님 준비 목록 — 사진·일러스트 스펙

> 공통: 배경 투명 PNG 또는 SVG. 앱 팔레트(크림 배경 #F7F5F0 위)에서 튀지 않게
> 라인 컬러는 잉크(#16140f)·파인 계열 권장. 폭 240px 이상이면 충분(그리드 셀용).

### 필수 (P5 비주얼 초이스) — 총 17컷
1. **허리 실루엣 3컷** — 강아지를 **위에서 내려다본** 몸통 실루엣:
   ① 잘록하게 들어감 ② 살짝 들어감 ③ 일자/볼록. 같은 강아지·같은 각도로 3단 변화만.
2. **배 라인 3컷** — **옆모습** 실루엣: ① 가슴에서 위로 올라감(턱업) ② 거의 일자
   ③ 아래로 처짐. 역시 동일 실루엣 3단 변화.
3. **갈비뼈 촉감 4컷** — 옆구리에 손을 얹은 그림: ① 갈비뼈가 드러나 보임
   ② 살짝 만지면 느껴짐(얇은 지방) ③ 꾹 눌러야 느껴짐 ④ 눌러도 안 느껴짐.
   단면도(지방층 두께)로 표현해도 좋음 — 스타일은 통일만 되면 됨.
4. **변 상태 7컷** — Bristol 1~7: 딱딱한 알갱이 → 이상적(촉촉한 소시지) → 물설사.
   ⚠️ 식품 앱이므로 실사 절대 금지, 부드러운 도형/라인 일러스트로. 색은 단색 브라운 계열.

### 권장 (있으면 바로 반영)
5. **주식 형태 3컷** — 그릇에 담긴 ① 건식사료 ② 화식(우리 실물!) ③ 반반.
   이건 **실사진 추천** — 우리 화식 실물이 들어가면 설문이 곧 제품 소개가 됨.
6. **케어 목표 5컷** — 체중/피부·털/관절/알레르기/일반 을 상징하는 미니 일러스트
   (현재 lucide 아이콘 대체, 리스트 버튼 좌측 40px 슬롯).

### 선택 (여유 있을 때)
7. **MCS 근육 4컷** — 등뼈·골반 위 근육 만지는 그림 4단계 (노령견만 보는 스텝이라 후순위).
8. **스텝 완료 마스코트 1~2컷** — 스텝 전환 때 살짝 지나가는 응원 컷 (브랜드 감성, 남용 금지).

## 5. 하지 않는 것 (명시)
- 웹 /start 라이트 설문은 이 기획 범위 밖 (앱 본설문만).
- 질문 삭제/추가 등 **데이터 스키마 변경 없음** — 표현·구조만. answers 키 전부 보존.
- 알고리즘(칼로리 v2·firstBox) 무접촉.
